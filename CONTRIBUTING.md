# Contributing to DocuMind Pro

Thank you for your interest in contributing to DocuMind Pro! We welcome contributions from the community to help improve this project.

## How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally:
    ```bash
    git clone https://github.com/your-username/documind-pro.git
    cd documind-pro
    ```
3.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/amazing-feature
    ```
4.  **Make your changes** and commit them with clear, descriptive messages.
5.  **Push your branch** to your fork:
    ```bash
    git push origin feature/amazing-feature
    ```
6.  **Open a Pull Request** on the main repository describing your changes.

## Development Setup

Please refer to the `README.md` for instructions on setting up the development environment.

## Style Guide

*   **Python**: Follow PEP 8 guidelines.
*   **JavaScript/React**: Use functional components and hooks. Keep styles clean and modular.
*   **Commits**: Use the [Conventional Commits](https://www.conventionalcommits.org/) specification (e.g., `feat: add new upload component`, `fix: resolve login error`).

## Reporting Issues

If you encounter any bugs or have feature requests, please open an issue in the repository's "Issues" tab. Provide as much detail as possible to help us understand and resolve the problem.
